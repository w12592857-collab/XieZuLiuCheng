# 章节生成规则

本文件规定每章从构思到正文的生成标准。

## 章节卡先行

每章正文生成前，必须先有章节卡。

章节卡保存目录：

`ZhangJieKa`

章节卡不是正文，不导出到小说正文里。

章节卡必须解决一个问题：这一章凭什么让读者继续看。

## 章节卡固定模板

每张章节卡至少包含：

1. 章节定位
2. 承接上一章
3. 本章一句话目标
4. 开场钩子
5. 本章危机
6. 危机来源
7. 主角目标
8. 系统选择
9. 奖励即时发放
10. 金手指破局
11. 当章爽点兑现
12. 伏笔和回收
13. 知情范围
14. 关键场面
15. 画面感要求
16. 结尾追读钩子
17. 避坑提醒
18. 字数目标

## 正文提示词规则

正文提示词必须优先依据章节卡。

优先级：

正文提示词 > 章节卡 > 小说总档案 > 大纲 > 旧稿

正文提示词必须把章节卡转成明确执行要求，不要只写泛泛建议。

必须包含：

- 写作定位
- 承接上一章
- 必须写出的剧情节点
- 开场钩子
- 危机与来源
- 金手指参与破局
- 当章爽点兑现
- 画面感要求
- 对话与情绪要求
- 知情范围
- 禁止写偏
- 结尾追读钩子

## 正文结构

推荐结构：

开场危机 -> 危机升级 -> 林夜判断 -> 金手指破局 -> 爽点兑现 -> 余波 -> 钩子

开场 300 字内尽量出现：

- 敌人动作
- 异常气息
- 人质风险
- 公开羞辱
- 舆论爆点
- 倒计时
- 规则触发
- 系统选择
- 身份危机

不要开场长篇复述上一章。

## 系统选择写法

如果本章出现系统选择，必须完整写出：

1. 触发原因
2. 三个选择
3. 林夜确认选择
4. 奖励立刻到账
5. 奖励如何参与本章破局
6. 奖励是否埋下后续伏笔

示例结构：

【选择三：当众斩断上弦再生链，奖励：再生链锁定。】
林夜没有犹豫。
“选择三。”
【选择确认。】
【奖励发放：再生链锁定已生效。】
下一息，他眼中的世界变了。堕姬和妓夫太郎之间那条肉眼看不见的血线，被太阳神火焰照得清清楚楚。

## 金手指调用规则

金手指必须解决具体危机，不要只写“主角很强”。

正确写法：

- 见闻色预判毒血轨迹，所以能救平民
- 太阳神火焰烧尽鬼血，所以毒无法扩散
- 日之呼吸配合再生链锁定，所以能同时斩断双鬼再生
- 剧情情报判断敌人不会本体下场，所以主角不被假象牵着走

错误写法：

- 林夜直接爆发更强力量
- 林夜随手解决所有敌人
- 林夜忽然学会新技能
- 敌人突然变蠢送死

## 战斗写法

用户明确不喜欢水战斗。

战斗要少写招式流水账，多写结果和反应。

推荐节奏：

危机压迫 -> 林夜看穿 -> 一击破局 -> 反派破防 -> 见证者反应 -> 结果落地

一场战斗里不要连续写三段以上“你来我往”的招式交换。除非每一段都带来新信息、新危机或新爽点。

## 爽点规则

每章至少兑现一个爽点。

爽点类型：

- 当众打脸
- 救下意难平角色
- 揭穿谎言
- 反杀布局
- 斩杀反派
- 获得原著强者认可
- 系统奖励到账
- 身份反转
- 敌人破防
- 群众认知反转

爽点前必须有压力。没有压力的胜利不够爽。

爽点后必须有余波。没有余波的爽点不够大。

## 字数规则

如果用户要求不超过 5000 字，默认从第二行正文开始统计非空白字符。

第一行章节标题不计入字数。

如果剧情超过字数上限，优先把后续余波、下一场战斗、反派会议、长解释放到下一章，不要硬塞。

## 正文格式

txt 正文：

- 第一行是章节标题
- 不使用 Markdown 标题层级
- 不留行间空行
- 不输出解释
- 不输出“下面是正文”

Markdown 资料：

- 章节卡、流程、审查报告、人物库、文风库可以使用 Markdown
- 资料文件不混入小说正文
